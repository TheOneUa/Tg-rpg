# TG-RPG v5.3.8

Фэнтези RPG для Telegram Mini App (и браузера).

## 📋 Changelog

### v5.3.8 (текущая)
**Исправления:**
- 🐛 При получении уровня больше не открывается экран статов автоматически — очки накапливаются, пульсирующая кнопка 🎒 в HUD напоминает о нераспределённых очках. Распределить можно в любой момент.

### v5.3.7 (предыдущая)
**Новое:**
- 🆕 **Квесты** — накапливаются и не пропадают. 24 квеста четырёх типов (убийства врагов, боссы, золото, глубина). Разные уровни сложности появляются по мере продвижения. Сдача квеста и получение награды (золото + опыт + предмет) — только у Старейшины.
- 🆕 **Торговец** — полноценная торговля: вкладка «Купить» (14 видов предметов) и «Продать» (всё: экипировка, зелья, ресурсы). Цена продажи = 40% от покупной.
- 🆕 **Мастера в домах** — три вкладки: «Статы» (6 характеристик включая ⚡ скорость атаки), «Улучшить» (предмет в слоте: +2 к бонусу), «Зачаровать» (5 видов зачарований: похищение жизни, крит, скорость, MP реген, HP).
- 🆕 **⚡ Скорость атаки** — новая характеристика: прокачивается у мастеров, падает с кольца, снижает кулдаун всех атак (макс −50%). Отображается в экране персонажа и в инвентаре.
- 🆕 **Экран персонажа** — фиксированный полноэкранный размер, вкладки не прыгают. Инвентарь с прокруткой, работает с любым количеством предметов.

### v5.3.6 (предыдущая)
**Новое:**
- 🆕 **Лут из монстров** — предметы выпадают прямо на пол рядом с убитым врагом. Обычные враги: 35% шанс. Боссы: 2-3 предмета гарантированно. Таблицы лута меняются с глубиной (с этажа 5 — редкие предметы).
- 🆕 **12 новых предметов**: мечи, луки, посохи, кольчуга, мантия, кольца — с бонусами к ATK/DEF/HP/MP/SPD. Редкость: common/rare/epic (подсвечивается цветом).
- 🆕 **Экипировка** — 3 слота: оружие, броня, кольцо. Бонусы суммируются и отображаются в экране персонажа.
- 🆕 **Экран персонажа** (кнопка 🎒 в HUD) — три вкладки: Статы, Экипировка, Инвентарь. Нажать на предмет в инвентаре — надеть. Кнопка «Снять» — вернуть в инвентарь.
- 🆕 Мечи и щиты удалены из quickbar и лута подземелья — заменены полноценной системой экипировки.
- 🆕 Золото теперь выпадает случайным количеством (10-30) вместо фиксированных 25.

### v5.3.5 (предыдущая)

### v5.3.4 (предыдущая)
**Исправления:**
- 🐛 **Кнопка паузы ☰ не отображалась** — инлайн-стиль `style="display:none"` в HTML перебивал CSS-класс `.visible { display: flex }`. Убран инлайн-стиль, управление только через класс.
- 🐛 **LoS не работал** — `getCurrentTM()` мог вернуть карту деревни вместо подземелья (зависит от `G.location`, а не `G.depth`). Теперь `_nearestEnemy` напрямую использует `G.dungeonGrid` при проверке видимости — без обхода через `getCurrentTM()`.
- 🐛 **A\* восстановление пути** — в старом `Pathfinding.js` было два блока восстановления пути, первый сломанный запускался раньше. Переписан чисто: один цикл `while (parent.has(rk))`, `parent` хранит ключ родителя (не объект).
- 🐛 **LoS конечная точка** — старый алгоритм возвращал `true` до проверки целевого тайла. Теперь сначала проверяется стена, потом возврат `true`.

### v5.3.1 (предыдущая)

### v5.0.4 (предыдущая)
**Исправления:**
- 🐛 Игра ускорялась/тормозила в зависимости от частоты обновления экрана (60 / 120 / 144 Hz) — вся логика была завязана на количество кадров `requestAnimationFrame`, а не на реальное время. Введён `delta time (dt)`, нормализованный к 60 FPS: скорость движения, кулдауны атак, таймеры частиц, текста урона, снарядов, стана врагов — теперь идут одинаково на любом устройстве независимо от FPS.
- 🐛 На iPhone (iOS Safari) нельзя было одновременно двигаться джойстиком и атаковать/пить зелье — срабатывало только одно касание. Кнопки атаки, магии, контекстного действия и слотов инвентаря переведены с `click` на `touchstart`, что даёт настоящую поддержку мультитача (джойстик одним пальцем + кнопка другим — работают параллельно).

**Изменения:**
- 🔧 Убрана надпись «Играй без авторизации» в главном меню — теперь при отсутствии Telegram-пользователя поле просто пустое.

## 🚀 Установка

Разархивируй в папку и открой `index.html` в браузере или задеплой на GitHub Pages / Telegram Mini App.

## 🖼️ Своё изображение на сплэш

Положи файл `assets/splash.png` (рекомендуется 360×360 px) — он автоматически появится на загрузочном экране.
